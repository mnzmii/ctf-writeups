## **Description**

The challenge presents a Python service that requires the user to send the hexadecimal byte `0xFF` three times, side-by-side, with no spaces. The service reads raw bytes from input and compares them to a specific byte sequence to reveal the flag.

---

## **Initial Thoughts**

- The source code reveals the use of `sys.stdin.buffer.readline()`, which indicates the program is handling **raw bytes** rather than standard UTF-8 text.
- Typing `0xFF` manually in a terminal sends the ASCII characters `'0'`, `'x'`, `'F'`, and `'F'`, which is 4 bytes per "character."
- The goal is to send the actual numerical value of **255** (the maximum value for a single byte) three times in a row.
- As a Computer Science student familiar with C++ and pointers, I recognize this as a test of understanding how data is represented in memory versus how it is represented in a text editor.

## **Step-by-Step Solution**

### **1. Analyze the Logic**

The provided script contains the following critical check:

Python

`if user_input == b"\xff\xff\xff":
    print(open("./flag.txt", "r").read())`

The `b` prefix denotes a **byte literal**. This means the server is expecting a 3-byte payload where each byte has the hex value `FF`.

### **2. The Keyboard Limitation**

Typing `0xFF` on a keyboard results in:

`0x30 0x78 0x46 0x46` (ASCII for '0', 'x', 'F', 'F')

This does not match the server's expectation of a single byte `0xFF`.

### **3. Crafting the Payload**

To send raw bytes, I used a Python one-liner to write directly to the `stdout` buffer. I added a `\n` (newline) because the server uses `readline()`, which waits for a newline character to terminate the input.

Bash

`python3 -c 'import sys; sys.stdout.buffer.write(b"\xff\xff\xff\n")' | nc lonely-island.picoctf.net 57157`

## **Vulnerability Analysis**

This challenge isn't a vulnerability in the traditional sense, but rather a demonstration of **Input Interpretation**.

- **The Concept:** Standard terminal input is often abstracted into characters. However, lower-level network services often expect raw binary data.
- **The Lesson:** Security professionals must distinguish between a **representation** of data (the text "0xFF") and the **actual** data (the byte value 255).

## **Lessons Learned**

- **Buffers vs. Strings:** In Python, `.buffer` allows for the manipulation of raw binary data, bypassing string encoding logic.
- **Piping Binary Data:** Tools like `printf` or `python -c` are essential for interacting with services that require non-printable characters.
- **Hex Representation:** Hexadecimal is just a human-friendly way to write binary; it is not the data itself
